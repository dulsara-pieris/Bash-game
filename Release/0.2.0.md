## âœ¨ What's New

### Modular Architecture
The game has been completely reorganized into 13 focused modules, each responsible for a specific aspect of the game:

#### Core Modules
- **config.sh** - All constants, colors, and terminal setup
- **utils.sh** - Helper functions (random numbers, ownership checks, exit handler)
- **profile.sh** - Player profile creation, loading, and saving

#### Game Systems
- **ships.sh** - Ship definitions and stats (5 ships)
- **skins.sh** - Skin customization system (5 skins)
- **entities.sh** - Asteroids, crystals, and powerups
- **weapons.sh** - Laser firing and combat mechanics
- **collision.sh** - Collision detection and game over handling
- **effects.sh** - Powerup timers and temporary effects

#### User Interface
- **render.sh** - All drawing functions (ship, HUD, borders, stars)
- **input.sh** - Keyboard input handling
- **menu.sh** - Main menu, stats, help, and update system
- **shop.sh** - Hangar and skin shop

#### Main Entry Point
- **game.sh** - Streamlined main game loop that orchestrates all modules

## ğŸ“ Notes

### Module Loading Order
Modules are loaded in dependency order:
1. config.sh (required by all)
2. utils.sh (required by many)
3. profile.sh (uses utils)
4. ships.sh & skins.sh (data definitions)
5. Remaining modules (game systems)

### Variable Scope
- Global variables used for game state (shared across modules)
- Local variables used within functions
- Clear documentation of which modules read/write which variables
